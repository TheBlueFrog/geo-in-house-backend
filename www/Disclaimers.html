GeoFence Research

Disclaimers

This app is mildly invasive of your privacy. All collected and uploaded data is labeled only by an installation GUID. That doesn't mean it's not identifiable, especially since the whole project is designed to identify you.

The chief goal is to decide if we can effectively use movement/location patterns to help out with identify problems.

Briefly, the device's physical activity is monitored at a set frequency (see the settings page) and at interesting changes in activity a location is obtained. This raw data is uploaded to the servers for analysis.

Based on the analysis GeoFences may be setup at interesting places, these can be downloaded back onto the device via the Download menu item.

All data is assumed to be freely given and all results are freely available.

Privacy

Some care has been put into how the data collected by this software is managed. There are two different aspects to this. First is the data collected for research purposes, what is collected is not well defined since it is continuously mutating and how it is used is ill-defined since that's why we are doing the research. As such I'm not saying much about it beyond there is some and it may be used, with normal safeguards.

The second kind of data is a bit different, to a large part it is fall out from the basic data collection however in an attempt to provide you, the users with some value, in the hopes of keeping you around. There are now some ways for devices to communicate geoinformation with each other. This has raised some issues about privacy, I'm following some basic rules to manage this

  • Stuff is tested inside my family before exanding to a wider audience. While I do have a bit of leverage because I pay the bills nothing gets out that's too big-brotherish.
  • Anything that leaves your device that will appear on another device is your data and you had to deliberately set it up so it leaves. By default nothing leaves.
  • The exception to the previous rule is the menu item "Ping Server", this event is sent to the servers and forwarded to all registered application instances. At this time you can't opt-out of this one, I plan on removing it soon-ish but it has proved to be invaluable for testing.
